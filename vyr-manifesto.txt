Vyr Manifesto
============

Correctness over cleverness.

Safe by default. Unsafe by declaration.

No hidden control flow. No hidden allocation. No hidden surprises.

If behavior matters, it must be visible.

Local reasoning is a feature, not a luxury.

Effects belong in signatures, not in guesswork.

Failing operations must be explicit and typed.

Contracts are first-class: preconditions, postconditions, invariants.

Code should be easy to review, hard to misuse, and stable under refactoring.

Small, coherent language beats feature pile-up.

One clear way is better than five clever ways.

General-purpose first, with real systems capability when needed.

Low-level control where it counts, without forcing low-level pain everywhere.

Multiple memory strategies, one safety story.

Ownership when you need precision. Regions when you need structure. GC when you need simplicity.

A standard runtime, with predictable profiles and no mystery costs.

Zero-cost abstractions when practical, never zero-clarity abstractions.

Performance must be strong and predictable, not accidental.

Tooling is not an afterthought; it is part of the language contract.

Fast incremental builds are a language feature.

Deterministic formatting and reliable refactoring are non-negotiable.

Compiler interfaces should be stable, versioned, and machine-readable.

Semantics must be precise, executable, and test-backed.

No folklore semantics.

No implementation inheritance as the default architecture trap.

No complexity for prestige.

No ambiguity that breaks humans or tools.

Portable by specification, not by undefined behavior.

Interop without surrendering safety.

Built for large codebases, long-lived systems, and teams that need trust.

Vyr: power with discipline.

Vyr: explicit by design.

Vyr: engineering over magic.
